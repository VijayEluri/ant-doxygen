Doxygen

Description

This task is a wrapper for the configuration options for doxygen, a document generation-tool. To know more about doxygen in the words of the author himself -

" Doxygen is a documentation system for C++, C, Java, IDL (Corba, Microsoft, and KDE-DCOP flavors) and to some extend PHP.

  1. It can generate an on-line documentation browser (in HTML) and/or an off-line reference manual from a set of documented source files. There is also support for generating output in RTF (MS-Word), PostScript, hyperlinked PDF, compressed HTML, and Unix man pages. The documentation is extracted directly from the sources, which makes it much easier to keep the documentation consistent with the source code.
  2. Doxygen can be configured to extract the code structure from undocumented source files. This can be very useful to quickly find your way in large source distributions. The relations between the various elements are be visualized by means of include dependency graphs, inheritance diagrams, and collaboration diagrams, which are all generated automatically."

Parameters

doxygen task does not take any attribute. But it takes 'property' child nodes as attribute. The structure of 'property' node is as follows:
       <property name="<doxygen_property_name>" value="<property_value>" />

The field 'doxygen_property_name' corresponds to the property defined as per the doxygen configuration file. Please see the following examples to get a better idea of the same.


Examples:

1. To generate documentation by default in current directory, basic option.

  • Ant 1.7+:

    <project name="my-project" xmlns:doxygen="antlib:org.doxygen.tools" .... />
    <doxygen:doxygen>
           <property name="INPUT" value="${inputdir}" />
    </doxygen:doxygen>

    • or

  • <taskdef name="doxygen" classname="org.doxygen.tools.DoxygenTask"
            classpath="${destdir}" />
    <doxygen>
           <property name="INPUT" value="${inputdir}" />
    </doxygen>

2. To generate html documentation recursively for a given dir. and suppress latex documentation

  • Ant 1.7+: <project name="my-project" xmlns:doxygen="antlib:org.doxygen.tools" .... />
    <doxygen:doxygen>
           <property name="INPUT" value="${inputdir}" />
           <property name="RECURSIVE" value="yes" />
           <property name="GENERATE_LATEX" value="false" />
           <property name="PROJECT_NAME" value="Ant-Doxygen Test Docs" />
           <property name="GENERATE_TREEVIEW" value="yes" />
           <property name="SOURCE_BROWSER" value="yes" />
    </doxygen:doxygen>
    • or
  • <taskdef name="doxygen" classname="org.doxygen.tools.DoxygenTask"
            classpath="${destdir}" />

    <doxygen>
           <property name="INPUT" value="${inputdir}" />
           <property name="RECURSIVE" value="yes" />
           <property name="GENERATE_LATEX" value="false" />
           <property name="PROJECT_NAME" value="Ant-Doxygen Test Docs" />
           <property name="GENERATE_TREEVIEW" value="yes" />
           <property name="SOURCE_BROWSER" value="yes" />
    </doxygen>
